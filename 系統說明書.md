## 1. 系統簡介

本平台協助企業進行溫室氣體盤查作業，支援：

- **範疇一（Scope 1）**：企業直接排放，例如固定燃燒源（天然氣、柴油）、移動燃燒源（汽油、車用柴油）、逸散排放（冷媒）。
- **範疇二（Scope 2）**：企業間接排放，例如外購電力、外購蒸汽。

系統自動依據排放係數計算 CO2e（公噸），並提供儀表板監控與 Excel 匯出功能。

## 2. 帳號登入

1. 開啟瀏覽器，進入系統首頁。
2. 輸入**帳號**與**密碼**，點選「登入」。
3. 登入成功後自動跳轉至儀表板。

| 角色 | 預設帳號 | 預設密碼 | 功能範圍 |
|------|----------|----------|----------|
| 管理員 | `admin` | `Admin@123` | 所有功能，含排程管理 |
| 一般使用者 | `user01` | `User@123` | 上傳、查看、匯出 |

> **注意：** 首次使用請立即修改預設密碼，避免安全風險。

## 3. 儀表板總覽

登入後首先看到儀表板，包含以下資訊：

| 區塊 | 說明 |
|------|------|
| **範疇一 CO2e 合計** | 所有範疇一原始數據計算後的 CO2e 總量（公噸） |
| **範疇二 CO2e 合計** | 所有範疇二原始數據計算後的 CO2e 總量（公噸） |
| **最近計算結果** | 最新 10 筆計算紀錄，含排放類別、項目、CO2e 值 |
| **報告進度燈號** | 各報告排程的目前狀態（綠燈 / 黃燈 / 紅燈） |

### 燈號說明

| 燈號 | 意義 |
|------|------|
| 綠燈 (Green) | 距離截止日期尚有充裕時間 |
| 黃燈 (Yellow) | 距離截止日期已達警示天數（預設 7 天內） |
| 紅燈 (Red) | 已超過截止日期，尚未完成數據上傳 |

## 4. Excel 數據上傳

這是系統的核心功能，用來匯入各類活動量數據並自動計算碳排放。

### 4.1 進入上傳頁面

點選上方導覽列的「**數據上傳**」進入上傳頁面。

### 4.2 Excel 欄位格式規範

上傳檔案（`.xlsx` 或 `.csv`）**第一列必須為標題列**，欄位名稱需與下表完全一致（區分大小寫）：

| 欄位名稱 | 必填 | 說明 | 範例值 |
|----------|------|------|--------|
| `Scope` | 是 | 排放範疇，只能填 `1` 或 `2` | `1` |
| `Category` | 是 | 排放類別（需與系統設定一致） | `固定燃燒源` |
| `ItemName` | 是 | 排放項目名稱（需與系統設定一致） | `天然氣` |
| `Value` | 是 | 活動量數值，必須大於 0 | `1500.00` |
| `Unit` | 否 | 活動量單位（若省略，系統填入「未知」） | `m³` |

> **重要：** `Category` 與 `ItemName` 的組合必須存在於系統排放係數設定中，否則該筆資料將儲存但不產生計算結果。

### 4.3 支援的排放類別與項目對照表

#### 範疇一（Scope = 1）

| Category（排放類別） | ItemName（排放項目） | 建議單位 |
|---------------------|---------------------|----------|
| 固定燃燒源 | 天然氣 | m³ |
| 固定燃燒源 | 柴油 | L |
| 固定燃燒源 | 液化石油氣 | L |
| 移動燃燒源 | 汽油 | L |
| 移動燃燒源 | 柴油(車用) | L |
| 逸散排放 | 冷媒R-22 | kg |

#### 範疇二（Scope = 2）

| Category（排放類別） | ItemName（排放項目） | 建議單位 |
|---------------------|---------------------|----------|
| 外購電力 | 台電電力 | kWh |
| 外購蒸汽 | 工業蒸汽 | MJ |

### 4.4 上傳步驟

1. 依照 4.2 格式準備 Excel 或 CSV 檔案。
2. 點選「**選擇檔案**」，選取準備好的檔案（最大 10 MB）。
3. 點選「**上傳**」按鈕。
4. 系統將逐列驗證並匯入：
   - 顯示「成功匯入 N 筆數據，並已完成 CO2e 計算」表示成功。
   - 若有錯誤，頁面下方會列出各列的錯誤原因，其餘有效列仍會正常匯入。

### 4.5 CO2e 計算公式

系統自動使用以下公式計算：

```
CO2e（公噸）= 活動量（Value）× 排放係數（Factor）× 全球暖化潛勢（GWP）÷ 1000
```

### 4.6 常見上傳錯誤

| 錯誤訊息 | 原因 | 解決方式 |
|----------|------|----------|
| 資料不完整或數值無效，已跳過 | Scope 為 0、Category/ItemName 為空，或 Value ≤ 0 | 確認該列所有必填欄位均有正確填寫 |
| 範疇值 X 無效（僅接受 1 或 2） | Scope 欄填了非 1/2 的數值 | 將 Scope 改為 `1` 或 `2` |
| 不支援的檔案格式 | 上傳了 .doc、.pdf 等不支援格式 | 改用 `.xlsx` 或 `.csv` 格式 |
| 檔案大小超過限制 | 檔案超過 10 MB | 分批上傳或壓縮數據量 |

## 5. 計算結果與資料匯出

### 5.1 查看計算結果

點選導覽列「**計算結果**」，可查看所有已計算的 CO2e 紀錄，包含：

- 排放範疇、類別、項目
- CO2e 總量（公噸）
- 上傳時間、計算時間、上傳者

### 5.2 匯出 Excel

頁面提供兩種匯出選項：

| 按鈕 | 匯出內容 | 檔案名稱格式 |
|------|----------|--------------|
| 匯出計算結果 | 含 CO2e 計算值的完整報表 | `ESG計算結果_YYYYMMDD_HHmm.xlsx` |
| 匯出原始數據 | 上傳的原始活動量數據 | `ESG原始數據_YYYYMMDD_HHmm.xlsx` |

## 6. 報告排程管理（Admin 限定）

此功能僅限**管理員（Admin）**帳號使用。

### 6.1 查看排程清單

點選導覽列「**報告排程**」，可查看所有已設定的報告排程，含燈號狀態。

### 6.2 新增排程

1. 點選「**新增排程**」。
2. 填寫以下欄位：

| 欄位 | 說明 | 範例 |
|------|------|------|
| 報告名稱 | 報告的完整名稱 | 月度碳排放盤查報告 |
| 頻率 | `Monthly`（月報）或 `Yearly`（年報） | Monthly |
| 負責窗口 | 負責人姓名或部門 | 環境管理部-王小明 |
| 提前警示天數 | 幾天前開始顯示黃燈 | 7 |
| 下次截止日期 | 下次報告的繳交截止日 | 2026-04-30 |

3. 點選「**儲存**」完成建立。

### 6.3 編輯 / 刪除排程

- 在清單頁面點選「**編輯**」可修改排程設定。
- 點選「**刪除**」會跳出確認頁面，確認後永久刪除該排程。

## 7. 常見問題

**Q：上傳後看不到計算結果，CO2e 顯示為 0？**
A：表示該筆資料的 `Category` + `ItemName` 組合在系統排放係數設定中找不到對應項目。請確認填寫的類別與項目名稱與第 4.3 節的對照表完全相符（注意全形/半形、空格差異）。

**Q：上傳 CSV 時亂碼？**
A：請確認 CSV 檔案以 **UTF-8** 編碼儲存。在 Excel 中另存新檔時，選擇「CSV UTF-8 (逗號分隔)」格式。

**Q：一次可以上傳多少筆資料？**
A：無筆數上限，但檔案大小不得超過 **10 MB**。

**Q：我上傳了錯誤的數據，可以刪除嗎？**
A：目前系統不提供前端刪除功能，請聯絡系統管理員協助從資料庫移除。

**Q：忘記密碼怎麼辦？**
A：請聯絡系統管理員重設密碼。
